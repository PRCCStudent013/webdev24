Insight

In the client-side web development course, I've learned to build interactive and dynamic user experiences using HTML, CSS, and JavaScript. This involves creating engaging interfaces and functionality that run directly in the user's web browser without the need for server-side processing.

Here are my completed projects for this course:

Project 5: Project 06-01

In this project, I programmed the actions of a sign-up form where users had to supply a username, email address, and password for a new account. Passwords had to be at least eight characters long and contain at least one letter and one number. As a validation test, users had to enter the password twice to confirm that they didn't mistype it. If the password didn't match the required pattern or if the two passwords weren't identical, the password field was flagged as invalid.

Project 6: Project 07-02

In this project, I generated a poker hand containing five cards randomly selected from a deck of cards. The names of the cards were stored in a text string that I converted into an array. To 'shuffle' the deck, I randomly sorted the array. Each time the user clicked a Deal button, the last five cards of the array were removed, reducing the size of the deck. When the size of the deck dropped to zero, a new randomly sorted deck was generated.

Project 8: Project 12-03

In this project, I used jQuery to add visual effects to a recipe page for the dessert website Save your Fork. Each recipe had an ingredients list and a list of directions. To save page space, both lists were initially hidden, but their contents could be revealed by clicking the Ingredients and Directions headings. When those headings were clicked, the lists toggled between sliding down to reveal their content and sliding up to hide it.
